C.A.R.E.S. — Conde Labac Residents System
Mobile Application Wireframes

Low-fidelity wireframes of the Flutter mobile application, derived from the Conde Labac Web System reference. Screens are grouped by user role: public / resident portal, citizen service flows, and the staff Management Information System (MIS) for Admin and Officer accounts. Figures are numbered for citation in the project paper.

Using these in your document: print this page to PDF (Ctrl+P — each section starts on a new page and figures never split), or screenshot individual figures and paste them into Word / Google Docs. Every figure carries its number, title, and annotation list so captions can be copied as text.
1.0

Navigation Flow Map

Routing mirrors the web system: visitors and Residents use the public portal, while Admin/Officer sign-ins are redirected to the MIS shell. Role-based access control (RBAC) hides staff modules from Residents entirely.

2.0

Authentication

Mobile equivalents of system.html (sign-in with role chips) and claim-account.html (3-step wizard).

3.0

Resident Portal

Mobile version of the public landing page (index.html) and resident portal. Four tabs: Home, Services, GIS Map, Profile.

4.0

Citizen Service Flows

The web's service modals become full-screen flows on mobile. Each submits to the shared in-app stores, so entries appear live in the staff MIS modules.

5.0

MIS — Admin / Officer

The web sidebar layout becomes an app bar + navigation drawer. Module list is filtered by role; the drawer, KPI grid, and card-list pattern repeat across all 11 modules.
